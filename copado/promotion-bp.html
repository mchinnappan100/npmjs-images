🚀 Copado Deployment Best Practices

📌 Context

During a recent deployment process, a new promotion record was created even though merge conflicts had already been resolved in a prior promotion. This led to duplication of effort and unnecessary rework.

⚠️ Problem Identified

  • New promotion record was created unnecessarily
  • Previously resolved merge conflicts had to be redone
  • Loss of time and duplication of effort

✅ Recommended Approach

  • Create a new promotion branch
  • Attach the branch to the existing promotion record
  • Reuse previously resolved merge conflicts
  • Avoid resetting deployment state

🧠 Why This Matters

In Copado, a promotion record acts as the container for deployment state, including resolved merge conflicts. Creating a new promotion record resets this state, which leads to repeated work.

By reusing the same promotion record and attaching a new promotion branch, teams can maintain continuity and efficiency in the deployment lifecycle.

📘 Best Practices

  • Always verify if a promotion record already exists before creating a new one
  • Use promotion branches to continue work instead of restarting deployments
  • Align team on deployment strategy before execution
  • Document merge conflict resolutions for traceability
  • Minimize creation of new promotion records unless absolutely necessary

🤝 Conclusion

Aligning on the agreed deployment approach ensures efficiency, avoids duplication, and saves valuable time. Proper use of promotion branches and promotion records is key to a streamlined Copado deployment process.
